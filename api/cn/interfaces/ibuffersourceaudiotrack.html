Interface IBufferSourceAudioTrack

直接通过音频数据源创建的音频流，继承于 LocalAudioTrack, 在此之上提供了控制音频数据处理流程的功能（开始/暂停/跳转等等）。

可以通过 AgoraRTC.createBufferSourceAudioTrack 来获取

Events

source-state-change

  • source-state-change(currentState: AudioSourceState): void

  • 当 BufferSourceAudioTrack 的音频数据源处理状态变化时，会触发此事件。

    Parameters

    • currentState: AudioSourceState

      一共有以下几种状态：

      • "stopped": 播放停止（可能时手动停止，有可能时播放结束）
      • "paused"： 播放暂停
      • "playing": 正在播放

    Returns void

Inherited from ILocalTrack.isMuted

TBD 标记该轨道是否被 mute

startProcessAudioBuffer

  • startProcessAudioBuffer(options?: AudioSourceOptions): void

  • 开始处理音频数据源

    开始处理音频数据源说明我们已经将音频数据送入我们的处理节点，如果发布了远端此时可以听到该音源的声音。 但是只有在调用 play 之后，我们才回将音频数据送入声卡，此时本地才能听到声音。

    Parameters

    • Optional options: AudioSourceOptions

      设置是否需要循环播放/开始播放的位置

    Returns void
